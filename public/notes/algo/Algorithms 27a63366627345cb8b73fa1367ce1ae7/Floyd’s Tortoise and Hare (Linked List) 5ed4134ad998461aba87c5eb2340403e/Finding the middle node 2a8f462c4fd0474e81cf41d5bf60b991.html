Finding the middle node

Fast and Slow pointers

  1. Use Cases:

This is used to find the middle of a data structure like linked list.

  1. Approach:

This algorithm uses two pointers: slow (üê¢) and fast (üêá) pointers

  • Both pointers start at the head
  • Slow pointer progresses by one in each iteration
    • slow = slow.next
  • Fast pointer progresses by two in each iteration
    • fast = fast.next.next
  • When fast reaches the end, slow will be at the middle
  • If fast becomes None or fast.next becomes None:
    • Break out of the loop

  1. Complexity

Worst case time complexity: O (n)

  • Worst case, slow == fast at the end of the linked list

Worst case space complexity: O (1)

  • Only uses constant space - slow and fast pointers

  1. Sample Code
slow = head
fast = head

while fast and fast.next:
	slow = slow.next
	fast = fast.next.next

return slow
